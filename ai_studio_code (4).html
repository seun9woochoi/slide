마크다운 프레젠테이션

개발자와 기술 문서 작성자를 위한 실무 교육

왜 마크다운으로 발표 자료를 만드는가?

  • 단순하고 널리 사용되는 구문으로 누구나 쉽게 작성 가능함.
  • 제목, 목록, 코드 블록, 표 등 필요한 대부분의 기능을 지원함.
  • Git 같은 버전 관리 시스템을 통해 변경 이력 추적이 용이함.
  • 코드가 포함된 기술 발표 자료 작성 시 코드 하이라이팅이 편리함.
  • 텍스트 기반이므로 다양한 포맷(HTML, PDF, PPTX)으로 변환이 자유로움.
이 슬라이드는 마크다운으로 프레젠테이션을 만드는 것의 장점을 소개합니다. 기존의 파워포인트나 키노트 같은 GUI 기반 툴과 비교했을 때, 마크다운은 텍스트 기반이라 Git으로 버전 관리가 쉽고, 특히 개발자들에게 친숙한 코드 블록이나 문법 강조를 손쉽게 추가할 수 있다는 점을 강조해 주세요.

주요 마크다운 프레젠테이션 도구

  • Pandoc: 다양한 포맷으로 변환하는 만능 문서 변환 도구.
  • Reveal.js: 기능이 풍부한 HTML 프레젠테ATION 프레임워크.
  • Slidev: 개발자를 위해 설계된 최신 프레젠테이션 도구.
  • Marp: 마크다운에 집중하여 슬라이드를 만드는 생태계.
  • 그 외 remark, reveal-md 등 다양한 CLI 기반 도구 존재.
여기서는 발표 자료 제작에 사용되는 주요 마크다운 기반 도구들을 소개합니다. 각 도구의 핵심적인 특징을 한 문장으로 요약해서 청중이 전체적인 그림을 그릴 수 있도록 도와주세요. Pandoc은 '변환기', Reveal.js는 '프레임워크', Slidev는 '최신 개발자용 툴', Marp는 '마크다운 중심 생태계'라는 키워드로 설명하면 이해하기 쉽습니다.

도구 선택 핵심 기준

무엇을 고려해야 하는가?

  • PDF 및 HTML 생성: 공유 및 발표를 위한 필수 출력 포맷 지원 여부.
  • 발표자 노트: 발표 시 참고할 수 있는 별도 노트 기능 제공 여부.
  • 확장성 및 커스터마이징: 재사용 컴포넌트 제작 및 테마 적용 가능성.
  • 목차(TOC) 생성: 긴 교육 자료에서 길을 잃지 않도록 돕는 기능.
  • 문서화 및 유지보수: 커뮤니티 활성화 및 지속적인 업데이트 여부 확인.
좋은 도구를 선택하기 위해 고려해야 할 핵심 기준들을 설명하는 슬라이드입니다. 특히 발표자 노트는 Reveal.js의 경우 'S' 키를 눌러 별도 창으로 열 수 있다는 구체적인 예시를 들어 설명하면 좋습니다. 청중이 자신의 요구사항에 맞춰 각 기준의 우선순위를 정할 수 있도록 안내해 주세요.

Pandoc + Reveal.js

가장 유연한 조합

  • Pandoc: 마크다운을 Reveal.js HTML 포맷으로 변환하는 역할 수행.
  • 다양한 HTML 프레젠테이션 포맷(Slidy, S5 등) 출력 가능.
  • Reveal.js의 모든 기능을 활용하기에는 제한적인 측면 존재.
  • 특정 기능을 위해 HTML 데이터 속성 설정이 어려움.
  • 커스터마이징과 확장성 측면에서 한계가 발생할 수 있음.
인포그래픽: Pandoc 변환 흐름
[slides.md]
[pandoc]
[index.html (Reveal.js)]
Pandoc은 강력한 문서 변환 도구지만, Reveal.js와 함께 사용할 때의 장단점을 명확히 설명합니다. 마크다운을 Reveal.js로 변환하는 기본 흐름을 보여주되, Reveal.js 고유의 HTML 데이터 속성을 마크다운 내에서 직접 제어하기 어렵다는 한계점을 지적해야 합니다. 이 조합은 표준 마크다운 문법으로 간단한 슬라이드를 만들 때 가장 효과적이라는 점을 알려주세요.

Pandoc + Reveal.js 실무 예시

명령어 한 줄로 슬라이드 생성

  • slides.md 라는 마크다운 파일 준비.
  • 커스텀 스타일을 위한 slides.css 파일 작성 가능.
  • 아래 명령어로 HTML 파일 생성 및 웹에서 소스 가져오기.
  • -t revealjs: 출력 포맷을 Reveal.js로 지정하는 옵션.
  • -V theme=serif: serif 테마를 적용하는 옵션.
예시: Pandoc 실행 명령어

pandoc -t revealjs -s -o index.html slides.md \
-V revealjs-url=https://unpkg.com/reveal.js/ \
--include-in-header=slides.css -V theme=serif
이 슬라이드에서는 Pandoc을 이용해 마크다운 파일을 Reveal.js 슬라이드로 변환하는 구체적인 명령어를 보여줍니다. 각 옵션의 의미를 간략히 설명해주세요. -t는 출력 타입, -s는 독립적인 HTML 파일 생성, -o는 출력 파일명, -V는 변수 설정, --include-in-header는 외부 CSS 파일을 포함시키는 역할을 합니다. 이 예시를 통해 청중이 직접 실습해볼 수 있도록 유도할 수 있습니다.

Reveal.js 단독 사용

HTML 프레임워크 직접 활용

  • 10년 이상 안정적으로 개발되어 온 성숙한 라이브러리.
  • 마크다운 파일을 내장 기능을 통해 직접 로드할 수 있음.
  • 풍부한 서드파티 플러그인 생태계 (필기, 다이어그램 등) 보유.
  • 발표자 노트를 위한 <aside class="notes"> 태그 지원.
  • 확장성이 부족하여 웹 컴포넌트 등 별도 해결책 필요 가능성.
예시: 마크다운 슬라이드 작성

<section data-markdown>
  <textarea data-template>
    ## 슬라이드 1
    첫 번째 슬라이드입니다.
    ---
    ## 슬라이드 2
    두 번째 슬라이드입니다.
  </textarea>
</section>
Reveal.js를 Pandoc 없이 단독으로 사용하는 방법을 설명합니다. HTML 파일 내에 data-markdown 속성을 이용해 마크다운 콘텐츠를 직접 넣거나 외부 파일로 로드할 수 있음을 보여주세요. Reveal.js의 가장 큰 장점은 오랜 역사와 안정성, 그리고 풍부한 플러그인 생태계라는 점을 강조합니다. 반면, 확장성 면에서는 커스텀 개발이 필요할 수 있다는 점도 언급해 주세요.

Marp

마크다운 중심의 슬라이드 생태계

  • --- 구분자를 사용하여 간단하게 페이지를 나눔.
  • VS Code 익스텐션을 통해 실시간 미리보기를 보며 편집 가능.
  • PDF, HTML, PPTX 파일로 내보내기 기능을 지원.
  • 레이아웃 직접 조절, 애니메이션 효과 등에는 한계 존재.
  • Marp CLI를 통해 커맨드 라인에서 변환 작업 수행 가능.
인포그래픽: Marp 사용 흐름
[slide.md]
[Marp for VS Code]
[실시간 미리보기]
[PDF/PPTX/HTML]
|
[Marp CLI]
[PDF/PPTX/HTML]
Marp의 특징을 소개하는 슬라이드입니다. `---`만으로 슬라이드를 나누는 단순함과 VS Code 익스텐션의 편리함을 강조하세요. 특히 PPTX로 내보낼 수 있다는 점은 다른 도구와 차별화되는 장점입니다. 하지만 복잡한 레이아웃이나 애니메이션에는 한계가 있다는 점도 솔직하게 전달해야 합니다. 사용 흐름을 인포그래픽으로 보여주어 GUI와 CLI 두 가지 방식을 모두 활용할 수 있음을 시각적으로 설명합니다.

Marp 커스텀 테마 만들기

VS Code에서 나만의 테마 적용

  1. 작업 폴더에 theme.css와 같은 CSS 파일 생성.
  2. CSS 파일 상단에 /* @theme your-theme */ 주석으로 테마 이름 정의.
  3. @import 'default'; 구문으로 기본 테마를 상속하고 스타일 덮어쓰기.
  4. VS Code 설정에서 Markdown > Marp: Themes 항목 검색.
  5. 생성한 theme.css 파일의 상대 경로를 설정에 추가.
예시: 커스텀 테마 CSS

/* @theme your-theme */
@import 'default';

h1 {
  color: red;
}
청중이 Marp를 더 잘 활용할 수 있도록 커스텀 테마 제작 방법을 안내합니다. 먼저 theme.css 파일을 만들고, @theme 주석으로 테마 이름을 지정한 뒤, VS Code 설정에 해당 파일 경로를 등록하는 과정을 설명합니다. 경로 설정 시에는 워크스페이스 기준의 상대 경로를 사용해야 한다는 점을 강조해주세요. 간단한 CSS 예시를 통해 청중이 직접 시도해볼 수 있도록 합니다.

Marp CLI 활용법

터미널에서 슬라이드 변환

  • npx를 사용하면 별도 설치 없이 최신 버전 즉시 사용 가능.
  • --pdf, --pptx 옵션으로 출력 포맷을 지정하여 변환.
  • -w 또는 --watch 옵션으로 파일 변경 시 자동으로 재변환.
  • -s 또는 --server 옵션으로 로컬 서버를 실행하여 슬라이드 제공.
  • PDF, PPTX, 이미지 변환 시 Google Chrome 등 브라우저 설치 필요.
명령어옵션설명비고
marp slide.md(기본)HTML 파일로 변환slide.html 생성
marp slide.md --pdf--pdfPDF 파일로 변환브라우저 필요
marp slide.md --pptx--pptxPPTX 파일로 변환브라우저 필요
marp -w slide.md-w파일 변경 감지 모드자동 새로고침 지원
marp -s ./slides-s로컬 서버 모드디렉토리 지정
Marp CLI의 핵심 기능을 소개합니다. npx를 통한 간편한 실행 방법, 주요 출력 포맷 변환 옵션, 그리고 watch/server 모드와 같은 개발 편의 기능을 설명합니다. 특히 PDF나 PPTX 변환에는 로컬에 크롬과 같은 브라우저가 설치되어 있어야 한다는 중요한 제약사항을 반드시 언급해야 합니다.

Slidev

개발자를 위한 차세대 프레젠테이션 도구

  • VueJS 컴포넌트를 마크다운 내에서 직접 사용하여 확장성 우수.
  • 발표자 모드, 문법 하이라이팅, LaTeX, 다이어그램 등 풍부한 기능 내장.
  • 카메라 화면 표시, 오디오 녹화 등 원격 세션에 유용한 기능 제공.
  • 개발 서버는 라이브 리로드(LiveReload) 기능을 지원하여 개발 편의성 높음.
  • 비교적 신생 도구이지만 매우 빠르게 성장하는 커뮤니티 보유.

사례 1: 교육 자료 제작

다일 간의 긴 교육 세션

  • 목표: 여러 날에 걸쳐 진행되는 기술 교육용 슬라이드 제작.
  • 고려사항: 수강생들이 길을 잃지 않도록 목차(TOC) 기능이 매우 중요함.
  • 선택: Fusuma 또는 Slidev. 두 도구 모두 목차 기능을 지원함.
  • Slidev는 자동 목차 생성을 지원하여 유지보수에 더 유리함.

사례 2: 빠른 프로토타이핑

아이디어 공유를 위한 신속한 문서화

  • 목표: 복잡한 설정 없이 빠르게 슬라이드 초안을 만들고 공유.
  • 고려사항: 설치 및 사용이 간편하고, PPTX 등 범용 포맷 출력이 필요함.
  • 선택: Marp. VS Code 익스텐션으로 즉시 시작 가능.
  • 프로토타입을 Marp로 만들고, 파워포인트로 마무리하는 워크플로우 가능.
예시: Marp 워크플로우
  1. VS Code에 Marp 익스텐션 설치.
  2. 마크다운으로 내용 작성 후 실시간 미리보기로 확인.
  3. --pptx 옵션을 주어 PPTX 파일로 변환.
  4. 동료에게 PPTX 파일을 공유하여 피드백 요청.
두 번째 사례로, 빠른 프로토타이핑 및 공유 상황을 제시합니다. 이 경우 Marp가 가장 적합한 도구임을 설명합니다. "프로토타입을 Marp로 만들고 파워포인트로 마무리하기" 워크플로우를 구체적인 단계로 보여주세요. VS Code 익스텐션의 간편함과 설치 없이 사용할 수 있는 npx 명령어는 '신속함'이라는 요구사항에 부합하는 강력한 장점입니다.

사례 3: 고도로 맞춤화된 발표

회사 브랜딩과 인터랙티브 기능 추가

  • 목표: 회사 그래픽 가이드에 맞춘 테마와 동적인 컴포넌트 적용.
  • 고려사항: 확장성과 슬라이드 커스터마이징이 가장 중요한 요소.
  • 선택: Slidev 또는 Reveal.js 직접 사용.
  • Slidev는 Vue 컴포넌트를 통해 재사용 가능한 동적 요소 제작 용이.
  • Reveal.js는 방대한 플러그인 생태계를 활용하거나 직접 플러그인 개발 가능.
도구확장 방식장점단점
SlidevVue 컴포넌트현대적이고 재사용성 높음Vue.js 학습 필요
Reveal.js플러그인 / JS강력하고 유연함직접 개발 시 복잡도 높음
Pandoc(제한적)간단한 CSS 수정 용이동적 기능 추가 어려움
고도의 커스터마이징이 필요한 사례를 다룹니다. 이 경우 확장성이 뛰어난 Slidev나 Reveal.js가 적합합니다. Slidev는 Vue 컴포넌트 기반의 현대적인 확장 방식을, Reveal.js는 전통적인 플러그인 방식을 제공한다는 점을 비교 설명합니다. Pandoc은 간단한 CSS 수정은 가능하지만 동적인 기능 확장에는 한계가 있다는 점도 함께 언급하여 각 도구의 확장성 수준을 명확히 구분해 줍니다.

발표자 노트 활용하기

청중에게 보이지 않는 나만의 대본

  • Reveal.js: <aside class="notes"> 태그 또는 data-notes 속성 사용.
  • Reveal.js (Markdown): Note: 또는 Notes: 같은 구분자 사용.
  • Marp: 발표자 노트는 PPTX로 내보낼 때 포함됨.
  • Slidev: 각 슬라이드 --- 구분자 아래에 노트를 작성하는 방식.
  • 발표자 뷰에서는 다음 슬라이드 미리보기와 경과 시간 등을 확인 가능.

최종 도구 선택 가이드

당신에게 맞는 도구는?

  • 단순함과 PPTX 호환성이 중요하다면 Marp를 추천함.
  • 최신 기능과 높은 확장성을 원한다면 Slidev가 좋은 선택.
  • 안정성과 풍부한 플러그인이 필요하다면 Reveal.js가 적합.
  • 다양한 포맷 변환이 주 목적이라면 Pandoc을 활용하는 것이 효율적.
